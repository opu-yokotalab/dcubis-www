■概論

D-Cubisは自由に簡易的な3D空間を作成でき、またマルチメディアコンテンツ(現在は静止画のみ)を3D空間内にさまざまなgimmick(仕掛け)を利用して配置したり、どこかのサイトへリンクさせたりすることで、
「自分だけの3D空間をWeb上で公開」できます。

「D-Cubis Official Site」(http://alpha.c.oka-pu.ac.jp/D-Cubis/)

D-Cubisは以下のファイルにより構成されます。

メタファイル( .xml ) このシステムの核となるファイル。
D-Cubisで利用する情報(空間、配置、コンテンツ)を示すファイル
空間情報( .bud) 3D空間(建物)の情報を記述するファイル
配置情報( .lad) マルチメディアコンテンツの配置、gimmickの配置、
リンクの配置などを指定するファイル
コンテンツ情報( .cod) マルチメディアコンテンツの情報を記述するファイル

また、D-Cubisは上記の情報を作成するソフトウェアとして下記のものがあります

空間情報作成ツール 3D空間を２次元で捉えて、
より簡単に空間情報を作成できるソフトウェア
コンテンツ情報作成ツール リンクや説明文などのコンテンツに関する情報を
付加しコンテンツ情報を作成するソフトウェア
配置情報作成ツール

3D空間にマルチメディアコンテンツを
ギミックを利用し配置するためのソフトウェア

　


■推奨動作環境


OS Windows 2000 以降　Macも動作確認済み
CPU Pentium Ⅲ (500MHz以上)
メモリ 256MB以上
インターネット接続 ADSL以上
主なテスト環境

Windows 2000 Professional、celeron 433MHz、メモリ192MB
(少し動きが遅くなるが問題はない)

※ このシステムは、扱う画像の枚数(200枚が限度)、サイズ、解像度によりマシン負荷の大きさが変わります
したがって、空間製作者はできるだけサイズを小さく(200×150くらい)、解像度も小さく(72dpi)
するように心がけてください。

画像の変換ツールの一例として、フリーソフトのGIMPなどは手軽に変換が可能です

    アニメーションGIFは利用できません。
